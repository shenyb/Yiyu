# 医研AI助手 — 产品方案 v0.2

> 面向非技术人员的 AI 智能助手，初始聚焦医生群体
> 用自然语言完成 PPT 制作、课题调研、文件问答等日常工作
> 给哥做的礼物，边学边做

---

## 一、产品定位

| 项目 | 内容 |
|---|---|
| **产品名称** | 医研AI助手（暂定） |
| **一句话描述** | 双击打开就能用的 AI 工作助手，说大白话就能做 PPT |
| **目标用户** | 哥（某医院医生，兼任教研室工作）→ 同事口碑传播 |
| **产品形态** | JavaFX 独立窗口，内嵌 Spring Boot 后端 |
| **分发方式** | 单文件解压包，双击 .bat 启动 |
| **商业模式** | 给哥的礼物，不收费 |

## 二、目标用户画像

以产品发起人的哥哥（某医院医生，兼任教研室工作）为典型用户：

- **技术背景**：零。不懂命令行、API、模型、参数
- **日常工作**：出门诊/值班 + 教研室讲课/带教
- **核心痛点**：
  - 做科室汇报 PPT 耗时（要整理内容、排版、找模板）
  - 知道 AI 能帮忙但现有工具门槛太高
- **学习意愿**：愿意用，但必须「双击就能用，说话就能干」
- **传播路径**：觉得好用 → 直接推给科室同事

## 三、MVP 功能范围

三个核心功能全部实现，保留原型中的三个入口卡片。

### 3.1 核心功能一：AI 做 PPT

**三种使用场景：**

| 场景 | 用户操作 | 说明 |
|---|---|---|
| 从零生成 | 输入主题，直接生成 | AI 基于自身知识生成大纲和内容 |
| 参考文档生成 | 输入主题 + 拖拽 PDF/DOCX 参考资料 | AI 读取参考文档，结合用户主题生成大纲 |
| 接续半成品 | 拖拽已有的半成品 .pptx | AI 读取现有 PPT 内容，在其基础上继续/完善/换风格 |

**标准流程（从零生成/参考文档生成）：**

| 步骤 | 用户操作 | 系统响应 |
|---|---|---|
| 1 | 打开软件，看到引导界面 | 展示场景卡片 |
| 2 | 点「做PPT」，输入主题（可附带拖拽参考文件） | AI 读取参考文件 + 生成大纲 |
| 3 | 查看大纲，提出修改意见 | 根据反馈调整大纲 |
| 4 | 确认大纲，点「生成」 | POI-TL 填充本地模板，生成 PPT 文件 |
| 5 | 点「打开文件」或「打开文件夹」 | Java 直接打开本地文件 |

**接续半成品流程：**

| 步骤 | 用户操作 | 系统响应 |
|---|---|---|
| 1 | 拖拽已有的 .pptx 到窗口 | Apache POI 读取现有 PPT 内容（文字+结构） |
| 2 | 描述需求（如"帮我加5页最新进展""换一个简洁风格""把第三页展开"） | AI 分析现有内容，生成补充/修改方案 |
| 3 | 确认方案，点「生成」 | POI 基于原 PPT 追加/修改内容 |
| 4 | 打开文件 | Java 直接打开本地文件 |

**技术方案**：Apache POI + POI-TL 模板填充
- 用户提供一份医院空模板（带 Logo/配色/字体）到本地模板目录
- AI 生成结构化内容（JSON 格式大纲）
- 参考文档场景：AI 读取 PDF/DOCX 全文后，结合内容生成大纲
- 半成品场景：Apache POI 读取现有 .pptx 的幻灯片文字内容，发给 AI 理解后继续加工
- POI-TL 将内容填充到模板占位符
- 保留原模板全部样式
- 生成的文件直接保存到用户指定目录，Java Desktop API 打开文件/文件夹

### 3.2 核心功能二：AI 课题调研

| 步骤 | 用户操作 | 系统响应 |
|---|---|---|
| 1 | 点「课题调研」，输入研究方向 | AI 联网检索文献 + 整理 |
| 2 | 查看调研报告框架 | 展示报告大纲（分章节） |
| 3 | 提出调整意见（加删章节、侧重方向） | 修改报告框架 |
| 4 | 确认，点「生成报告」 | 生成 Word 文档 |
| 5 | 打开文件 | Java 直接打开本地文件 |

**技术方案**：DeepSeek API（联网搜索能力）
- 用户输入研究方向，AI 联网检索中文 + 英文文献
- 整理为结构化调研报告
- 支持上传参考文献（PDF），AI 参考后生成报告
- 输出格式：DOCX（Apache POI）

### 3.3 核心功能三：文档问答

| 步骤 | 用户操作 | 系统响应 |
|---|---|---|
| 1 | 点「读文件」，拖拽或选择文件 | AI 读取文件内容 |
| 2 | 输入问题 | AI 基于文件内容回答 |
| 3 | 追问 | 保持上下文继续回答 |
| 4 | 导出问答记录 | 生成摘要文档 |

**技术方案**：Apache PDFBox + Tika 解析文件，DeepSeek 理解回答
- 支持的格式：PDF、DOCX、TXT
- 从本地目录读取文件，不涉及上传
- 单轮问答 + 多轮对话上下文保持

### 3.4 MVP 不做

- 多用户/权限系统
- 语音输入（后续可加）
- 手机端
- 病历数据接入（安全合规考虑）
- 知识库 RAG（后续可加）

## 四、交互设计原则

### 4.1 核心原则：引导式对话，不要空白输入框

```
❌ 糟糕的设计：
   [________________________________] 发送

✅ 好的设计：
   推荐场景卡片（做PPT）
   点击卡片自动填充提示语
   用户改几个字就能发送
```

### 4.2 关键交互流程

**做PPT（三种入口）**

```
① 欢迎页 → 点击「做PPT」卡片
② 对话框自动填入引导语，用户补充具体需求（主题+要求）
③ 可选：拖拽参考资料（PDF/DOCX），AI 参考后生成大纲
④ 可选：拖拽半成品 .pptx，AI 读取后继续完善
⑤ AI 生成大纲，以结构化卡片呈现
⑥ 用户可点击「修改」「加一节」「满意，生成」
⑦ 生成进度条 → 完成 → 「打开文件」「打开文件夹」
```

**课题调研**

```
① 欢迎页 → 点击「课题调研」卡片
② 输入研究方向（如"查一下ADC药物在肺癌中的进展"）
③ AI 联网检索文献 → 生成报告大纲
④ 用户调整框架（加章节、改侧重）
⑤ 用户可拖拽上传参考文献 PDF
⑥ 确认 → 生成 DOCX 报告 → 「打开文件」
```

**读文件**

```
① 欢迎页 → 点击「读文件」卡片
② 拖拽或选择本地 PDF/DOCX/TXT
③ AI 读取文件，展示摘要
④ 用户输入问题 → AI 回答
⑤ 可追问，保持上下文
⑥ 可导出问答记录
```

### 4.3 非技术化措辞原则

| 不用 | 改用 |
|---|---|
| 模型、Token、参数 | （不提） |
| 生成内容、输出 | 帮您做、给您整理 |
| 确认/提交 | 好的，就这样 / 再改改 |
| 错误/失败 | 没成功，换个说法试试？ |
| 系统提示 | （直接用对话语气） |

## 五、技术方案

### 5.1 总体架构

```
┌──────────────────────────────────────────────────────┐
│  用户桌面 — 独立窗口                                  │
│                                                       │
│  start.bat（双击运行）                                 │
│    ↓                                                  │
│  jlink 裁剪的 JRE + JavaFX（~40MB）                    │
│    ↓                                                  │
│  JavaFX 窗口（WebView 渲染 Vue 3 前端）                │
│    ↓                                                  │
│  内嵌 Spring Boot 3 后端（随机端口）                    │
│    ├ Spring AI（DeepSeek API 调用）                    │
│    ├ Apache POI + POI-TL（PPT 模板填充）               │
│    └ 本地文件读写（直接 new File()）                    │
│                                                       │
│  ┌──────────────────────────────────────────────┐     │
│  │ DeepSeek API（云端）                          │     │
│  │ API Key 写在 application.yml，不 commit       │     │
│  └──────────────────────────────────────────────┘     │
└──────────────────────────────────────────────────────┘
```

### 5.2 技术栈明细

| 层 | 技术 | 说明 |
|---|---|---|
| 桌面容器 | JavaFX WebView | 内嵌 Chromium 内核，零成本，开源 |
| 后端框架 | Spring Boot 3 + JDK 21 | Java 生态 |
| 前端 | Vue 3（npm 项目模式） | Vite 构建，打包为静态资源 |
| AI 接入 | Spring AI + DeepSeek API | 统一抽象，可切换模型 |
| PPT 生成 | Apache POI + POI-TL | 模板填充方案 |
| 打包分发 | jlink 裁剪 JRE + JavaFX jmods → .bat | 单目录，解压即用，~50MB |
| 文件操作 | Java Desktop API + 本地文件 I/O | 直接读写用户本地文件 |

### 5.3 API Key 策略

- 写在 application.yml 或 .env 文件中
- .gitignore 排除，不 commit
- Key 用哥自己的 DeepSeek 账号，或者我先充值 ¥100 兜底
- 同科室推广时换成我统一管理

### 5.4 数据安全

- 不涉及 HIS/病历等内网数据
- 用户上传的模板文件存在本地
- 生成的文件存在用户指定目录
- 所有文件操作不走网络

## 六、PPT 模板方案（POI-TL）

### 6.1 工作流程

```
医生提供一个带医院 Logo 的空模板 .pptx → 放到软件 templates/ 目录
                ↓
开发者在模板中标记占位符 {{title}} {{content}} 等
                ↓
AI 根据用户需求生成结构化内容（JSON）
                ↓
POI-TL 将数据填入模板占位符
                ↓
输出最终 PPT 到用户指定目录
                ↓
用户点击「打开文件」→ Java Desktop.open() 直接打开
```

### 6.2 示例数据模型

```json
{
  "pptTitle": "糖尿病足的诊疗新进展",
  "subtitle": "科室业务学习",
  "date": "2026年6月",
  "slides": [
    {
      "type": "title",
      "title": "糖尿病足的流行病学"
    },
    {
      "type": "content",
      "title": "发病率与危险因素",
      "items": [
        "全球糖尿病足溃疡年发病率约 6.3%",
        "主要危险因素：周围神经病变、外周动脉疾病",
        "..."
      ]
    }
  ]
}
```

## 七、打包与分发

### 7.1 打包结构

```
medical-ai-assistant/
├── runtime/               # jlink 裁剪的 JRE + JavaFX (~40MB)
├── app.jar                # Spring Boot + Vue 3 静态资源
├── templates/             # 模板目录（用户放 .pptx）
├── output/                # 生成文件默认保存位置
├── application.yml        # DeepSeek API Key 等配置
└── start.bat              # 双击启动
```

### 7.2 start.bat

```bat
@echo off
chcp 65001 >nul
start javaw -jar app.jar
```

### 7.3 jlink 命令（参考）

```bash
jlink --module-path jdk-modules:javafx-jmods-21 \
  --add-modules java.base,java.desktop,java.sql,\
javafx.controls,javafx.web,jdk.unsupported,jdk.crypto.ec \
  --strip-debug --compress 2 --no-header-files --no-man-pages \
  --output medical-ai-runtime
```

## 九、后续扩展方向

- **语音输入**：减少打字门槛
- **知识库 RAG**：上传科室指南、教材，AI 基于本地知识回答
- **更多行业模板**：教师课件、银行报告等
- **移动端**：微信小程序查看/分享
- **多人协作**：团队共享模板和知识库
- **私有化部署**：医院内网环境，数据不出院
